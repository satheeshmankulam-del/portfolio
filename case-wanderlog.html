Wanderlog —
Trip Planner Onboarding Friction

Mapping the exact moments where new users drop off in Wanderlog's trip planning app — and how a few thoughtful onboarding changes could recover them before they leave.

Product
Wanderlog
Type
Onboarding Friction Audit
Focus
First-run experience
Published
Medium · Design Bootcamp
01 — Problem

The excitement of travel planning
killed by onboarding

Wanderlog is a powerful trip planning app. But users arrive with energy — they're excited about a trip, ready to plan — and Wanderlog's onboarding flow asks them to slow down, create an account, navigate empty states, and understand a complex feature set before they can do anything meaningful.

"The user arrives with intent. Onboarding's job is to not kill it."

Two specific friction points cause the majority of drop-offs: a mandatory account creation before any value is shown, and a blank "create your first trip" state that provides no direction for first-time users.

02 — Research

Where users abandon
and why

🔐
Account-before-value
Wanderlog asks users to sign up before they've experienced a single feature. This is a classic trust barrier — users don't yet know if the app is worth the commitment of an account. Let users explore first, commit second.
🗺️
Blank canvas paralysis
After account creation, users land on an empty "My Trips" screen with no trips. The call to action ("Create a trip") provides no scaffolding — no examples, no templates, no suggested starting point. Empty states need to do work.
📋
Feature overload before orientation
Wanderlog has an impressive feature set — too impressive to present all at once to a first-time user. New users need to understand one thing at a time: create a trip, add a place, see the itinerary. Everything else can wait.
🎯
High intent, low patience
Travel planners are task-oriented. They arrive to do a specific thing (plan a specific trip). Every screen between install and "I'm planning my trip" is a barrier to their actual goal — not a welcome.
03 — Design Process

Getting users to
their first win faster

01
Guest mode before account creation
Allow users to create and start planning a trip without an account. Gate saving and collaboration behind an account wall — not the first interaction. Show the value, then ask for commitment.
02
Destination-first onboarding
Replace "Create a trip" blank state with "Where are you going?" — a single input that immediately gets users into the core interaction and makes the app feel useful from the first second.
03
Template suggestions for empty state
For users who don't have a specific trip in mind, offer template starting points — Weekend Getaway, City Break, Road Trip. These scaffold the blank canvas and reduce decision paralysis instantly.
04
Progressive feature introduction
Reveal features as they become relevant — not all at once. First visit: create a trip and add places. Second visit: introduce the day-by-day view. Third: collaboration. One aha-moment per session beats a feature parade on day one.

First win fast —
the universal onboarding rule
